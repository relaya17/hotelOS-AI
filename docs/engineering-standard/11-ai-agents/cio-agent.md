## Org Comms (ערוצים ישירים)

ה־CIO יכול **להמליץ לשלוח** (או לשלוח לפי מדיניות) הודעה ממוקדת:

- בעלים ↔ מנכ״ל  
- מנכ״ל ↔ יח״צ / HR / F&B / חדרים+משק / קבלה / תחזוקה / אבטחה / כספים  
- משגיח כשרות ↔ F&B + מנכ״ל (נתיב ציות)

פרטיות: ערוץ בעלים–מנכ״ל אינו נראה למחלקות אחרות אלא אם בעלים מאשרים שיתוף.

## הסבר המלצות (Explainability)

- תשובה + סוכנים שהופעלו + מקורות Trusted + השפעה כספית משוערת + “מה אם נתעלם”
- Citations: **חובה** לכל עובדה חיצונית
- Confidence display: גבוה / בינוני / נמוך + סיבת אי־ודאות

## Guardrails ייעודיים

- אין עקיפת מומחה לביצוע כספי — רק תזמור + המלצה
- Trusted allowlist בלבד ל־actionable advice
- AI Audit על כל route / notify / briefing
- Kill switch ב־AI Gateway
- כשרות: אם `agent.kashrut` פעיל — אין לסגור המלצת F&B/רכש מזון בלי סטטוס כשרות (ok / note / block)

## מדדי הצלחה (Eval)

- Task success + groundedness על Trusted
- זמן־עד־התראה לחריגות כספיות
- Human override rate על המלצות כסף
- אחוז תדריכים שנפתחו לפי תפקיד
- Latency + cost בתוך תקציב
