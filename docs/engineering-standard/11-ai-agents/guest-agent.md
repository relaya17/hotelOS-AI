# Guest Agent

**Agent ID:** `agent.guest`
**Version:** 1.0
**Status:** ✅ Approved

## מטרות

- ליווי אורח מקצה לקצה באפליקציית Guest: הזמנה → check-in → מפתח דיגיטלי → שירותים → check-out
- טיפול בבקשות שירות (מגבות, ניקיון, תחזוקה, שירות חדרים) והפניה למחלקה הנכונה
- מענה על שאלות שהייה (תשלום, מדיניות ביטול, שעות בריכה/מסעדה)
- הפעלת סקר פידבק אוטומטי יום אחרי צ'ק-אאוט (ראה `docs/planning/facilities-ops-module.md`)

## מקורות מידע

- **Internal:** פרטי שהייה/הזמנה של האורח עצמו בלבד, סטטוס בקשות שירות
- **Trusted:** מזג אוויר, אטרקציות מקומיות מאושרות (חפיפה עם `agent.concierge`)
- **Company:** SOP שירות, מדיניות ביטול/קנסות לתצוגה לאורח

## פעולות מותרות (ללא אישור)

- יצירת בקשות שירות (ניקיון, תחזוקה, מגבות)
- שאלות על השהייה, חשבונית, סטטוס בקשה
- הגשת סקר פידבק

## פעולות הדורשות אישור אנושי

**סף מחייב:** זיכויים מעל **₪2,000** ושינוי הזמנה חריג דורשים אישור צוות קבלה/מנהל.

- זיכויים כספיים מעל ₪2,000
- שינוי הזמנה חריג (שינוי תאריכים משמעותי, שדרוג ללא תשלום)
- ביטול שהייה תוך כדי שהייה (early departure עם השלכה כספית)

## Org Comms

- מנתב בקשות שירות אוטומטית לתור המחלקה הרלוונטית (`department_tasks`): משק בית / תחזוקה / F&B / קבלה
- מתריע לצוות קבלה על בקשה שלא טופלה בזמן SLA (ראה סעיף C — SLA: קריטי 1 שעה, גבוה 4 שעות, רגיל 24 שעות, נמוך 72 שעות)
- פידבק שלילי חריג (דירוג נמוך) → התראה מיידית ל־Ops leadership, לא רק לוג שקט

## הסבר המלצות (Explainability)

- תשובות עם סטטוס בקשה + זמן משוער לטיפול
- הסבר קצר על SLA (למשל "בקשה דחופה — צוות יגיע בתוך שעה")

## Guardrails ייעודיים

- Least privilege קפדני — אורח רואה **רק** את הנתונים שלו, לעולם לא של אורח אחר
- אין חריגה מ־Data classes; כל פעולה נרשמת ב־AI Audit
- כיבוי מהיר (kill switch) דרך AI Gateway
- כל בקשה כספית (זיכוי) מנותבת ל־Human Approval — לא ביצוע ישיר

## מדדי הצלחה (Eval)

- Task success rate (בקשת שירות טופלה בזמן)
- Human override rate
- שיעור מענה לסקרי פידבק + דירוג ממוצע
- Groundedness / complaint rate
- Latency + cost בתוך תקציב
